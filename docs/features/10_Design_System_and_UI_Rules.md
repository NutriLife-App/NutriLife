# NutriLife — Design System and UI Rules

## 1. Мета
Зафіксувати базові візуальні та UX-правила NutriLife для MVP, щоб інтерфейс був цілісним, впізнаваним і стабільним при подальшій реалізації.

## 2. Product intent
NutriLife має виглядати як сучасний nutrition-first продукт, а не як generic fitness app, медична система чи технічний прототип. Користувач повинен відчувати спокій, чистоту, турботу і практичність.

## 3. Візуальний напрям
- light-first
- full dark mode support
- premium wellness aesthetic
- м’які зелені акценти
- теплі нейтральні відтінки
- rounded cards
- комфортні відступи
- спокійний контраст

## 4. Візуальні принципи
- чистий простір важливіший за декоративність
- інтерфейс має бути повітряним, але не пустим
- однакова логіка карток, кнопок і полів по всьому продукту
- мінімум візуального шуму
- світла тема є основною бренд-подачею

## 5. Theme strategy
### 5.1 Light
- основний вигляд продукту
- використовується в презентаціях, демо, бренд-комунікації
- має давати відчуття чистоти, читабельності та довіри

### 5.2 Dark
- повноцінно підтримується
- не виглядає як вторинна або поламана тема
- має бути комфортною, але не надто контрастною

### 5.3 Theme behavior
- базовий варіант для MVP: follow system
- у Settings закладається місце для System / Light / Dark

## 6. Компоненти
### 6.1 Buttons
- primary button
- secondary button
- ghost / tertiary action

Primary:
- для головної дії на екрані
- лише одна головна кнопка на ключовий екран

Secondary:
- для допоміжної дії

Ghost:
- для дрібних дій типу Back, Change, Placeholder action

### 6.2 Cards
Використовувати для:
- summary
- plan blocks
- recipe cards
- grocery sections
- settings sections

Правила:
- чіткі внутрішні відступи
- помірне заокруглення
- легке візуальне відділення від фону

### 6.3 Inputs
- зручні на мобільних
- читабельний текст
- error state
- focus state
- не надто низькі по висоті

### 6.4 Chips
Використовувати для:
- tags
- selectable options
- dietary restrictions
- activity level
- goals

## 7. Типографіка
- велика читабельність
- чітка ієрархія заголовків
- короткі підзаголовки
- не зловживати дрібним текстом

Рівні:
- screen title
- section title
- body text
- caption / helper text

## 8. Відступи
Використовувати єдину spacing scale.
Принцип:
- один модуль — одна логіка відступів
- секції відокремлюються повітрям, а не хаосом

## 9. Localization and copy rules
- українська є primary locale
- англійська є secondary locale
- user-facing strings не хардкодяться в компонентах
- копірайт має бути коротким, дружнім і не інфантильним
- пояснення додаються тільки там, де реально знімають тертя

## 10. UX rules
- на кожному екрані має бути зрозуміло, що робити далі
- важливе рішення не ховати
- форми не повинні виглядати карально
- навігація має бути прогнозованою
- не створювати тупиків без явного виходу

## 11. Що уникати
- строкатих кольорів
- надлишку іконок
- спортивної агресивної естетики
- блискучого “фітнес-маркетингу”
- надміру щільних екранів
- випадкових веб-компонентів у mobile UI

## 12. Acceptance criteria
- усі екрани виглядають як частини одного продукту
- кнопки мають послідовну ієрархію
- картки мають однакову логіку
- світла тема цілісна
- темна тема цілісна
- інтерфейс читається легко і без шуму
- тексти легко локалізуються

## 13. Notes for implementation
- цей документ є джерелом правди для colors.ts, spacing.ts та reusable UI components
- будь-який новий screen повинен узгоджуватися з цими правилами
